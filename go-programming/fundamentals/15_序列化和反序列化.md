# 序列化和反序列化

## 关于 JSON 数据

<span style="color:#9951FF">JSON</span>（JavaScript Object Notation）是一种<span style="color:#9951FF">轻量级数据交换格式</span>，既易于人类阅读与编写，也便于机器解析与生产。RESTful API 接口通常以 JSON 格式返回数据。

基础 JSON 格式：

```json
{
  "a": "Hello",
  "b": "World"
}
```

复杂 JSON 格式：

```json
{
  "result": [
    {
      "_id": "59f6ef443ce1fb0fb02c7a43",
      "title": "笔记本电脑"
```

## 结构体与 JSON 序列化

为 App 或小程序开发后端 API 接口时，通常需要在 Go 结构体与 JOSN 数据之间进行相互转换。

在 Go 中，**JSON 序列化**指将结构体数据转换为 JSON 格式字符串，**JSON 反序列化**则是将 JSON 数据解析为对应结构体对象。

这一过程主要通过标准库 `encoding/json` 中的 `json.Marshal`（序列化）和 `json.Unmarshal()`（反序列化）函数实现。若对性能有更高要求，也可使用 API 完全兼容标准库的第三方库 `github.com/json-iterator/go`，其执行效率更优。

### 结构体对象转换为 JSON 字符串

```go
package main

import (
	"fmt"

	json "github.com/json-iterator/go"
)

type Student struct {
	Id     int
	Gender string
	name   string
	Sno    string
}

func main() {
	s := &Student{
		Id:     1,
		Gender: "女",
		name:   "平泽唯",
		Sno:    "s0001",
	}
	fmt.Printf("%#v\n", s)

	jsonByte, _ := json.Marshal(s)
	jsonStr := string(jsonByte)
	fmt.Println(jsonStr)
}
```

> [!tip]
>
> **未导出（私有）字段不能参与 JSON 序列化与反序列化**，原因是 `enconding/json` 包作为外部包，没有权限访问结构体中未导出的成员。

### JSON 字符串转换为结构体对象

```go
package main

import (
	"fmt"

	json "github.com/json-iterator/go"
)

type Student struct {
	Id     int
	Gender string
	Name   string
	Sno    string
}

func main() {
	jsonStr := `{"Id":1, "Gender":"女", "Name":"平泽唯", "Sno":"s0001"}`
	s := Student{}
	err := json.Unmarshal([]byte(jsonStr), &s)

	if err != nil {
		fmt.Println(err)
	}

	fmt.Printf("%#v\n%v\n", s, s.Name)
}
```

> [!tip]
>
> `json.Unmarshal` 第二个参数为 `any`（即 `interface{}`），故可通过 `map[string]any` 接收反序列化结果。

## 结构体标签 Tag

结构体 Tag 是<span style="color:#A82323">字段元信息</span>，可在运行时<span style="color:#A82323">通过反射读取</span>，定义于字段类型后，由反引号包裹。

```go
key1:"value1" key2:"value2"
```

每个 Tag 由<span style="color:#A82323">一个或多个键值对</span>组成，遵循以下规则：

- 键与值之间用冒号（`:`）分隔。
- 值必须用双引号（`"`）包裹。
- 多个键值对之间以空格分隔。

Tag 的解析对格式要求<span style="color:#A82323">极为严格</span>：若书写不符合规范（如键值间加空格、遗漏双引号、用单引号代替双引号等），编译器不会报错，运行时也无异常抛出，但<span style="color:#A82323">通过反射无法正确解析该 Tag</span>，会导致依赖 Tag 的功能（如 JSON 序列化）失效。

错误示例：

```go
`json: "name"`   // 键与值之间存在多余空格，导致解析失败
```

正确示例：

```go
`json:"name"`
```

标准库（如 `encoding/json`）及第三方库（如 `GORM`、`validator`）均依赖结构体 Tag，因此 **Tag 格式必须准确无误**。

```go
package main

import (
	"encoding/json"
	"fmt"
)

type Student struct {
	Id     int    `json:"id"`
	Gender string `json:"gender"`
	Name   string `json:"name"`
	Sno    string `json:"sno"`
}

func main() {
	s := &Student{
		Id:     1,
		Gender: "女",
		Name:   "平泽唯",
		Sno:    "s0001",
	}
	fmt.Printf("%#v\n", s)

	jsonByte, _ := json.Marshal(s)
	jsonStr := string(jsonByte)
	fmt.Println(jsonStr)
}
```

## 嵌套结构体和 JSON 序列化及反序列化

```go
package main

import (
	"fmt"

	json "github.com/json-iterator/go"
)

// 学生
type Student struct {
	Id     int
	Gender string
	Name   string
}

// 班级
type Class struct {
	Title    string
	Students []Student
}

func main() {
	c := &Class{
		Title:    "0302",
		Students: make([]Student, 0, 200),
	}

	for i := range 10 {
		s := Student{
			Name:   fmt.Sprintf("stu_%02d", i),
			Gender: "女",
			Id:     i,
		}
		c.Students = append(c.Students, s)
	}

	data, err := json.Marshal(c)

	if err != nil {
		fmt.Println("JSON marshal failed.")
		return
	}

	fmt.Printf("JSON: %s\n", data)
}
```
