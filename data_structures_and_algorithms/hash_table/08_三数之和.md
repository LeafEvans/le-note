# 三数之和

## 题目

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。请你返回所有和为 `0` 且不重复的三元组。

> [!tip]
>
> 答案中不可以包含重复的三元组。

**示例 1**：

```sh
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。
```

**示例 2**：

```sh
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。
```

**示例 3**：

```sh
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。
```

**提示**：

- `3 <= nums.length <= 3000`
- `-10⁵ <= nums[i] <= 10⁵`

## 思路

<img src="../../images/data_structures_and_algorithms/image_20251106_223500.gif"  />

以 `nums` 数组为例，首先对数组进行**排序**。然后，使用一层 `for` 循环，从索引 `i = 0` 开始，同时定义指针 `left` 指向 `i + 1`，指针 `right` 指向数组末尾。

此时，目标是找到 `a`、`b`、`c` 使得 `a + b + c == 0`，其中 `a = nums[i]`、`b = nums[left]`、`c = nums[right]`。

接下来，开始移动指针：

- 若 `nums[i] + nums[left] + nums[right] > 0`，说明三数之和大（因为数组一排序），所以将 `right` 向左移动，以减小和值。
- 若 `nums[i] + nums[left] + nums[right] < 0`，说明三数之和小，所以将 `left` 向右移动，以增大和值。

Go 代码如下：

```go
import "sort"

func threeSum(nums []int) [][]int {
	sort.Ints(nums)
	result := make([][]int, 0)
	n := len(nums)
	for i := 0; i < n-2; i++ {
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}
		left, right := i+1, n-1
		for left < right {
			sum := nums[i] + nums[left] + nums[right]
			if sum == 0 {
				result = append(result, []int{nums[i], nums[left], nums[right]})
				for left < right && nums[left] == nums[left+1] {
					left++
				}
				for left < right && nums[right] == nums[right-1] {
					right--
				}
				left++
				right--
			} else if sum < 0 {
				left++
			} else {
				right--
			}
		}
	}
	return result
}
```

- 时间复杂度：`O(n²)`
- 空间复杂度：`O(1)`

## 总结

一般而言，考虑 `a`、`b`、`c` 三个数的去重，其对应的就是 `nums[i]`、`nums[left]`、`nums[right]`。

### `a` 的去重

**`a` 若重复了怎么办？**

`a` 是 `nums` 中遍历的元素，因此应该直接跳过。

**但应该判断 `nums[i] == nums[i + 1]` 还是 `nums[i] == nums[i - 1]` 呢？**

若使用前者，有：

```go
if nums[i] == nums[i+1] {
  continue
}
```

此时就将三元组中出现重复元素的情况也省略了。例如 `[-1, -1, 2]` 这组数据，当遍历到第一个 -1 时，判断下一个也为 -1，那么这组数据就被省略了。

此题要判断的是**不能有重复的三元组**，而**三元组内的元素可重复**。

```go
if i > 0 && nums[i] == nums[i-1] {
  continue
}
```

此时就是遍历 `nums[i]` 时，判断前一位是否为一致的元素，例如 `[-1, -1, 2]` 这组数据，当遍历到第一个 -1 时，只要前一位没有 -1，那么 `[-1, -1, 2]` 这组数据可以收录到结果集中。

因为数组已排序，因此当固定第一个数 `nums[i]` 时，所有以它开头的三元组都会被找出来。若下一个数与它相同，所有以它为开头的组合，必然已经被前一个数找过了。

### `b` 与 `c` 的去重

当前的 `left` 和 `right` 已构成一个有效三元组，若 `left+1` 位置的值和 `left` 相同，那么再用它作为 `b`，配合同样的 `a` 和某个 `c`，一定会产生重复三元组。

同理适用于 `right`。

因此需要跳过所有与当前 `b` 或 `c` 值相同的相邻元素，确保下一次 `(b, c)` 是“新”的。

## 思考

若三数之和可以使用双指针法，则两数之和是否可以使用双指针法？

两数之和不可使用双指针法，其要求返回的是索引下标，而双指针法一定要排序，一旦排序后原数组的索引就被改变了。

若两数之和要求返回的是数值，则可以使用双指针法。
