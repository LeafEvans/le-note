# RESTful API 测试和调试

无论是开发新接口或者对接第三方服务，高效的测试与调试能力都是后端/全栈开发者的基本功。

## 常用测试工具

### Postman

Postman 是业内最流行的图形化 API 测试工具，支持环境变量、自动化脚本和接口文档生成。

**基本操作流**：

1. 创建新请求（Request）
2. 选择 HTTP 动词（GET / POST / PUT / DELETE）
3. 填入 API 端点（URL）
4. 配置请求头（Headers）与请求体（Body）
5. 点击 Send 并检查响应状态及数据

**Postman Collection（集合）导出示例**：

你可以将一组 API 导出 JSON 格式，方便团队共享：

```json
{
  "info": {
    "name": "用户管理 API 测试",
    "description": "测试用户相关的所有 API 端点"
  },
  "item": [
    {
      "name": "获取用户列表",
      "request": "GET",
      "url": {
        "raw": "{{baseUrl}}/api/users?page=1&limit=10",
        "host": ["{{baseUrl}}"],
        "path": ["api", "users"],
        "query": []
      }
    }
  ]
}
```

### cURL

在没有图形界面的 Linux 服务器上，`curl` 是测试接口的最快方式。

```bash
# GET：获取用户列表
curl -X GET "https://api.example.com/users" \
		 -H "Content-Type: application/json"

# POST：创建新用户（-d 后面跟请求体数据）
curl -X POST "https://api.example/com/users" \
	   -H "Content-Type: application/json" \
	   -d '{
	   	"name": "张三",
	   	"email": "zhangsan@example.com"
	   }'
	   
# PUT：更新用户信息
curl -X PUT "https://api.example.com/users/123" \
		 -H "Content-Type: application/json" \
		 -d '{
		 	"name": "李四",
		 	"email": "lisi@example.com"
		 }'

# DELETE: 删除特定用户
curl -X DELETE "https://api.example.com/users/123" \
		 -H "Content-Type: application/json"
```

> [!tip]
>
> - `-X` 指定方法
> - `-H` 添加请求头
> - `-d` 发送请求体数据

### 浏览器开发者工具（Console）

在进行前端检查时，直接在浏览器的 Console 面板中使用 `fetch` 发起请求是非常直观的做法。

```javascript
// 在浏览器控制台中执行
fetch('https://api.example.com/users', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
})
  .then(response => response.json())
  .then(data => console.log('✓ 请求成功:', data))
  .catch(error => console.error('✕ 请求失败:', error));
```

## 常见问题排查与调试

### CORS 跨域问题

**典型的报错信息**：

```bash
Access to fetch at 'https://api.example.com' from origin 'https://localhost:3000' has been blocked by CORS policy...
```

**根本原因**：浏览器的同源策略限制了前段跨域访问。

**解决方案**：必须由**服务端**配置允许跨域的 Header（以 Node.js/Express 为例）：

```javascript
// 服务端全局中间件配置 CORS
app.use((req, res, next) => {
  // 允许所有域名访问（生产环境建议改为具体的白名单域名）
  res.header('Access-Control-Allow-Origin', '*');
  // 允许的请求方法
  res.header('Access-Control-Allow-Methods', 'GET, PUT, POST, DELETE, OPTIONS');
  // 允许携带的请求头（如 Token）
  res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  
  // 处理预检请求 (Preflight)
  if (req.method === 'OPTIONS') {
    return res.sendStatus(200);
  }
  next();
});
```

### 状态码排雷指南

看到报错先看 HTTP 状态码，能帮你省下 80% 的瞎猜时间：

| 状态码  | 常见原因                    | 解决思路                                                     |
| ------- | --------------------------- | ------------------------------------------------------------ |
| **400** | 请求数据格式错误/缺少必填项 | 检查 Body 的 JSON 格式、参数类型是否与接口文档一致。         |
| **401** | 身份验证失败                | 检查 Header 中的 Token 是否缺失、过期或拼写错误。            |
| **404** | API 路径不存在              | 确认 URL 拼写、动态路由参数（如 `/users/undefined`）是否正确。 |
| **500** | 服务器内部代码崩溃          | 前段无需改动，直接去服务端看终端控制台或错误日志报错。       |

### JSON 数据格式陷阱

严格的 JSON 规范要求非常苛刻，千万别把 JS 对象字面量直接当 JSON 发送：

```json
// ✕ 错误的格式（导致 400 Bad Request）
{
  name: "张三",                 // 错误：键名必须用双引号
  'email': 'test@example.com', // 错误：不能使用单引号
  age: "25",                   // 不规范：age 应为数字类型，不需要引号
  role: "admin",               // 错误：最后一项末尾不能有逗号 (Trailing comma)
}

// ✓ 标准的 JSON 格式
{
  "name": "张三",
  "email": "test@example.com",
  "age": 25,
  "role": "admin"
}
```

## 自动化测试进阶

手动点 Postman 很容易遗漏边界情况，企业级项目通常会引入自动化测试（如 Jest, Mocha, PyTest）。

### 单元/集成测试示例（基于 Jest）

```javascript
// user.api.test.js
describe('用户 API 自动化测试', () => {
  
  test('GET /api/users - 应该成功获取用户列表', async () => {
    const response = await fetch('http://localhost:3000/api/users');
    const result = await response.json();
   
    // 断言（Assertions）
    expect(response.status).toBe(200);           // 状态码应为 200
    expect(result.success).toBe(true);           // 业务状态应为 true
    expect(Array.isArray(result.data)).toBe(true); // data 必须是数组
  });
 
  test('POST /api/users - 应该成功创建新用户', async () => {
    const payload = {
      "name": "自动化测试账号",
      "email": "auto@test.com"
    };
   
    const response = await fetch('http://localhost:3000/api/users', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload)
    });
   
    const result = await response.json();
   
    // 断言
    expect(response.status).toBe(201);            // 创建成功应返回 201
    expect(result.success).toBe(true);
    expect(result.data.name).toBe(payload.name);  // 校验返回的数据是否一致
    expect(result.data).toHaveProperty('id');     // 数据库应该自动生成了 ID
  });
  
});
```
