# RESTful API 实际应用

我们以最常见的“用户管理系统”为例，串联前面学过的所有规范，设计一套工业级的 RESTful API。

## 路由设计

在一个标准系统中，所有的路由端点（Endpoints）应保持高度的统一和语义化。

```http
### 核心用户操作
GET    /api/users				# 列表：获取用户列表（支持分页/搜索）
GET    /api/users/{id}  # 详情：获取特定用户
POST   /api/users       # 创建：注册新用户
PUT    /api/users/{id}  # 更新：修改用户信息
DELETE /api/users/{id}  # 删除：注销用户

### 嵌套资源操作（用户与订单）
GET  /api/users/{id}/orders  # 列表：获取某用户的所有订单
POST /api/users/{id}/orders  # 创建：为某用户下新订单
GET  /api/orders/{orderID}   # 详情：获取特定订单（扁平化查询）
```

## 标准的 CRUD 报文

### 创建用户（Create）

创建成功时，规范的 HTTP 状态码应返回 `201 Created`。

```http
// → POST /api/users
{
	"name": "王五",
	"email": "wangwu@example.com",
	"phone": "1380000000"
}

// ← 201 Created
{
	"success": true.
	"message": "用户创建成功",
	"data": {
		"id": 125,
		"name": "王五",
		"email": "wangwu@example.com",
		"createdAt": "2024-01-15T10:00:00Z"
	}
}
```

### 读取用户（Read）

读取列表时携带分页参数，读取成功返回 `200 OK`。

```http
// → GET /api/users?page=1&limit=5&sort=createdAt&order=desc

// ← 200 OK（列表带分页结构）
{
	"success": true,
	"data": [
		{"id": 125, "name": "王五", createdAt: "2024-01-15T10:00:00Z"},
		{"id": 124, "name": "李四", createdAt: "2024-01-15T09:00:00Z	"}
	],
	"pagination": {
		"currentPage": 1,
		"totalPages": 3,
		"totalItems": 15,
		"itemsPerPage": 5
	}
}
```

### 更新用户（Update）

整体替换用户信息，成功返回 `200 OK`。

```http
// → PUT /api/users/125
{
	"name": "王五",
	"email": "wangwu.new@example.com"
}

// ← 200 OK 
{
	"success": true,
	"message": "用户信息更新成功",
	"data": {
		"id": 125,
		"name": "王五",
		"email": "wangwu.new@example.com",
		"updatedAt": "2024-01-15T11:30:00Z"
	}
}
```

### 删除用户（Delete）

成功删除后，返回 `200 OK`（包含提示）或 `204 No Content`（无返回体）。

```http
// → DELETE /api/users/125

// ← 200 OK
{
	"success": true,
	"message": "用户删除成功"
}
```

## 典型返回错误

### 数据校验失败（如邮箱格式错误）

前段传参有误，返回状态码 `400 Bad Request`，并在 `details` 中精准指出错误字段。

```json
// ← 400 Bad Request
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "数据验证失败",
    "details": [
      {"field": "name", "message": "姓名不能为空"},
      {"field": "email", "message": "邮箱格式不正确"}
    ]
  }
}
```

### 资源不存在

请求了数据库中没有的 ID，返回状态码 `404 Not Found`。

```json
// ← 404 Not Found
{
  "success": false,
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "用户不存在"
  }
}
```

## 客户端调用示例（JS）

前段使用 `Fetch API` 调用上述标准 RESTful 接口的经典封转模版：

```javascript
// 获取用户列表（GET）
async function getUsers() {
    try {
        const response = await fetch('/api/users');
        const result = await response.json();

        result.success
            ? console.log('用户列表:', result.data)
            : console.error('获取失败:', result.error.message);
    } catch (error) {
        console.error('网络错误:', error);
    }
}

// 创建新用户（POST）
async function createUser(userData) {
    try {
        const response = await fetch('/api/users', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(userData)
        });

        const result = await response.json();
        result.success
            ? console.log('用户创建成功:', result.data)
            : console.error("创建失败:", result.error.message);
    } catch (error) {
        console.error('网络错误:', error);
    }
}

// 删除用户（DELETE）
async function deleteUser(userId) {
    try {
        const response = await fetch(`/api/user/${userId}`, { method: 'DELETE' });
        const result = await response.json();

        result.success
            ? console.log('用户删除成功')
            : console.error('删除失败:', result.error.message);
    } catch (error) {
        console.error('网络错误:', error);
    }
}
```
