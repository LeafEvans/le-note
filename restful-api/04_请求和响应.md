# RESTful API 请求与响应

## JSON 规范

现代 RESTful API 几乎全部采用 **JSON**（Javascript Object Notation）作为数据交互格式。它是互联网数据的“通用语言”，轻量、跨语言且人类易读。

```json
{
  "name": "张三",
  "age": 25,
  "hobbies": ["阅读", "游泳"],
  "address": {
    "city": "北京",
    "district": "朝阳区"
  },
  "isActive": true
}
```

## 交互示例

客户端与服务端的交互，核心在于**请求头（Headers）**的声明和**请求体（Body）**的传递。

### GET 请求（获取数据，无请求体）

```http
// → Request（告诉服务器：我期望接收 JSON）
GET /api/users/123 
Accept: application/json

// ← Response
{
	"id": 123,
	"name": "张三",
}
```

### POST 请求（提交数据，有请求体）

```http
// → Request (告诉服务器：我发给你的是 JSON)
POST /api/users 
Content-Type: application/json

{
	"name": "李四",
	"password": "secure123"
}

// ← Response
{
	"id": 124,
	"name": "李四",
	"message": "用户创建成功"
}
```

## 统一响应结构

在实际工程中，服务端**绝对不能**直接返回裸数据。为了让前段能够统一拦截和处理，所有响应都必须包裹在一个**标准的“外壳”**中。

### 成功响应

包含明确的状态标识（`success` 或 `code`）、提示信息和核心数据。

```json
{
  "success": true,
  "message": "操作成功",
  "data": {
    "id": 123,
    "name": "张三"
  },
  "timestamp": "2024-01-15T08:30:00Z"
}
```

### 错误响应

发生错误时，`data` 为空，必须提供明确的错误码，并在必要时给出具体的错误细节（如表单校验失败的字段）。

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "输入数据验证失败",
    "details": [
      {
        "field": "email",
        "message": "邮箱格式不正确"
      }
    ]
  },
  "timestamp": "2024-01-15T08:30:00Z"
}
```

### 分页响应

返回列表数据时，必须单开一个 `pagination` 字段，告诉客户端当前数据页码及总数。

```json
{
  "success": true,
  "data": [
    {"id": 1, "name": "用户1"},
  	{"id": 2, "name": "用户2"}
  ],
	"pagination": {
    "currentPage": 1,
    "totalPages": 10,
    "totalItems": 100,
    "itemsPerPage": 10
  }
}
```

## 常用请求头

请求头（Headers）是 HTTP 报文的“元数据”，用于携带控制信息。以下是 API 开发中最核心的 4 个请求头：

| 字段名          | 核心作用                                | 常见示例                        |
| --------------- | --------------------------------------- | ------------------------------- |
| `Content-Type`  | 声明**发送给服务器**的数据格式          | `application/json`              |
| `Accept`        | 声明**期望服务器返回**的数据格式        | `application/json`              |
| `Authorization` | 携带用户的**身份凭证**（Token）         | `Bearer token123`               |
| `User-Agent`    | 标识**客户端的类型**（如浏览器/App 版） | `MyApp/1.0` 或 `Mozilla/5.0...` |
