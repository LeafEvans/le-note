# HTTP 方法

## HTTP 方法

HTTP 方法就像是客户端发送给服务器的**动作指令**。

在 RESTful API 架构中，最核心的四种方法完美对应了数据库的 **CRUD（增删改查）**操作。

| HTTP 方法 | 核心作用 | 对应 CRUD    | 生活类比（餐厅点餐）       |
| --------- | -------- | ------------ | -------------------------- |
| GET       | 获取资源 | Read（查）   | 拿菜单，看看今天有什么菜   |
| POST      | 创建资源 | Create（增） | 告诉服务员，我要点一份菜   |
| PUT       | 更新资源 | Update（改） | 换菜（把之前的菜整盘换掉） |
| DELETE    | 删除资源 | Delete（删） | 告诉服务员，取消这道菜     |

### GET 方法（获取）

**GET 用于向服务器索取数据。**

```http
// 获取所有用户列表
GET /api/users

// 获取 ID 为 123 的特定用户
GET /api/users/123

// 获取该用户的订单流水
GET /api/users/123/others
```

- **安全性**：只读操作，绝不会修改或破坏服务端的数据。
- **可缓存**：浏览器和代理服务器默认会缓存 GET 请求，提升性能。
- **参数可见**：参数通常拼接在 URL 中传递（如 `?name=张三`），有长度限制且不适合传敏感信息。

### POST 方法（创建）

**POST 用于向服务器提交新数据。**

```http
// 创建一个新用户
POST /api/users
{
	"name": "张三",
	"email": "zhangsan@example.com"
}

// 创建一笔新订单
POST /api/orders
{
	"userId": 123,
	"items": ["商品A", "商品B"]
}
```

- **修改状态**：会在服务器上产生实质性的数据新增。
- **非幂等性**：**连续发送两次相同的 POST 请求，会创建出两个不同的资源**（下两单就会上两份菜）。
- **参数隐蔽**：核心数据封装在请求体（Body）中，安全性较高，且无文件大小限制。

### PUT 方法（更新）

**PUT 用于更新现有的整个资源。**

```http
// 整体更新 ID 为 123 的用户信息
PUT /api/users/123
{
	"name": "李四",
	"email": "lisi@example.com",
	"phohe": "13800138000",
}
```

- **全局替换**：通常用于将目标资源**整体覆盖替换**为请求体中的新数据。
- **幂等性**：**连续发送一百次相同的 PUT 请求，结果和发送一次完全一样**（无论覆盖多少次，最终订单状态是一样的）。
- **兜底机制**：根据 HTTP 规范，如果请求更新的资源不存在，PUT 也可以被用来直接新建该资源。

### DELETE 方法（删除）

**DELETE 用于销毁服务器上的指定资源。**

```http
// 删除指定用户
DELETE /api/users/123

// 删除指定订单
DELETE /api/orders/456
```

- **销毁资源**：彻底删除服务器上的指定数据。
- **幂等性**：删除一次和删除一百次，最终结果都是“该资源已不存在”，不会尝试副作用。

## HTTP 状态码

当服务器处理完你的“动作指令”后，会通过**状态码**向你汇报执行结果。状态码由三位数字组成，按首数字分为不同的逻辑类别：

| 状态码  | 英语短语                | 归属类别             | 含义说明                                                     |
| ------- | ----------------------- | -------------------- | ------------------------------------------------------------ |
| **200** | `OK`                    | **✓ 2xx 成功**       | **请求成功**。服务器已完美接收并处理。                       |
| **201** | `Created`               | **✓ 2xx 成功**       | **创建成功**。常作为 POST 请求的专属成功回应。               |
| **400** | `Bad Request`           | **✕ 4xx 客户端错误** | **请求报文有误**。如参数缺失、JSON 格式错误，服务器拒绝处理。 |
| **401** | `Unauthorized`          | **✕ 4xx 客户端错误** | **未授权**。你需要先登录，或提供的 Token 无效。              |
| **404** | `Not Found`             | **✕ 4xx 客户端错误** | **资源不存在**。你请求的 URL 地址或数据找不到。              |
| **500** | `Internal Server Error` | **💥 5xx 服务端错误** | **服务器内部崩溃**。代码抛出未捕获的异常或数据库挂了。       |
